🚀 Enhanced TP/SL System - ระบบ TP/SL ขั้นสูงใหม่!

🎯 การเลือกวิธีการคำนวณ TP/SL

ใช้พารามิเตอร์ InpTPSLMethod ในการเลือกวิธีการคำนวณ TP/SL ที่ต้องการ:

0 = FIXED_POINTS
1 = ATR_BASED
2 = SWING_POINTS
3 = PERCENTAGE_BASED
4 = HYBRID_METHOD

💡 แนะนำ: F1 Basic ใช้ 0-3, F2 Gold ใช้ 1-2, F3 Pro ใช้ 4

📊 Fixed Points Method (วิธีแบบเดิม)

เหมาะสำหรับมือใหม่

คำอธิบาย: กำหนด TP/SL เป็นจำนวน Points ตายตัว ไม่เปลี่ยนแปลงตามสภาวะตลาด

ข้อดี: เข้าใจง่าย ควบคุมได้ ไม่ซับซ้อน

ข้อเสีย: ไม่ปรับตามความผันผวนของตลาด

พารามิเตอร์ คำอธิบาย ค่าแนะนำ หมายเหตุ
InpTakeProfitPoints Take Profit เป็น Points 3500.0 ทอง / 500 สำหรับ Forex
InpStopLossPoints Stop Loss เป็น Points 2000.0 ทอง / 300 สำหรับ Forex

💡 ตัวอย่างการตั้งค่า:

InpTPSLMethod = 0  // Fixed Points
InpEnableTPSL = true
InpTakeProfitPoints = 500.0    // 50 pips สำหรับ Forex
InpStopLossPoints = 300.0      // 30 pips สำหรับ Forex

📈 ATR-Based Method (ปรับตามความผันผวน)

เหมาะสำหรับระดับกลาง

คำอธิบาย: ใช้ Average True Range (ATR) ในการคำนวณ TP/SL ให้ปรับตามความผันผวนของตลาด

ข้อดี: ปรับตามสภาวะตลาด รองรับความผันผวน มีเหตุผล

ข้อเสีย: ซับซ้อนกว่า อาจให้ SL/TP ที่กว้างเกินไปในบางครั้ง

พารามิเตอร์ คำอธิบาย ค่าแนะนำ ช่วงที่เหมาะสม
InpATRTimeframe กรอบเวลาสำหรับคำนวณ ATR PERIOD_H4 H1/H4/Daily/Weekly
InpATRPeriod จำนวนแท่งเทียนสำหรับ ATR 14 10-21 แท่ง
InpATRMultiplierSL ตัวคูณ ATR สำหรับ Stop Loss 2.0 1.5-3.0
InpATRMultiplierTP ตัวคูณ ATR สำหรับ Take Profit 3.0 2.0-5.0

💡 ตัวอย่างการตั้งค่า:

InpTPSLMethod = 1  // ATR-Based
InpATRTimeframe = PERIOD_H4
InpATRPeriod = 14
InpATRMultiplierSL = 2.0      // SL = ATR × 2
InpATRMultiplierTP = 3.0      // TP = ATR × 3

การคำนวณ: หาก ATR = 0.0050, SL = 0.0100, TP = 0.0150

🎯 Swing Points Method (ใช้จุด High-Low)

เหมาะสำหรับระดับกลาง

คำอธิบาย: หาจุด Highest High และ Lowest Low ย้อนหลังเป็นจำนวนแท่งที่กำหนด แล้วใช้เป็นฐานในการคำนวณ TP/SL

ข้อดี: ใช้โครงสร้างตลาดจริง เหมาะกับ Swing Trading

ข้อเสี่ยง: อาจให้ SL/TP ที่กว้างมากในตลาดที่มี Range กว้าง

พารามิเตอร์ คำอธิบาย ค่าแนะนำ ช่วงที่เหมาะสม
InpSwingTimeframe กรอบเวลาสำหรับหาจุด Swing PERIOD_H4 H1/H4/Daily
InpSwingLookback จำนวนแท่งย้อนหลังสำหรับหา High/Low 20 10-50 แท่ง
InpSwingBufferPercent เผื่อระยะห่างเป็นเปอร์เซ็นต์ 10.0 5-20%

💡 ตัวอย่างการตั้งค่า:

InpTPSLMethod = 2  // Swing Points
InpSwingTimeframe = PERIOD_H4
InpSwingLookback = 20         // มอง 20 แท่งย้อนหลัง
InpSwingBufferPercent = 10.0  // เผื่อ 10%

การคำนวณ: หา High/Low ใน 20 แท่ง แล้วเผื่อระยะ 10%

📊 Percentage-Based Method (เปอร์เซ็นต์)

เหมาะสำหรับมือใหม่

คำอธิบาย: กำหนด TP/SL เป็นเปอร์เซ็นต์ของราคาเข้า เหมาะสำหรับการจัดการ Risk แบบเปอร์เซ็นต์

ข้อดี: ควบคุม Risk ได้ชัดเจน เข้าใจง่าย

ข้อเสีย: ไม่คำนึงถึงสภาวะตลาด อาจไม่เหมาะกับทุกสถานการณ์

พารามิเตอร์ คำอธิบาย ค่าแนะนำ ช่วงที่เหมาะสม
InpSLPercentage Stop Loss เป็นเปอร์เซ็นต์ของราคาเข้า 2.0 1.0-5.0%
InpTPPercentage Take Profit เป็นเปอร์เซ็นต์ของราคาเข้า 4.0 2.0-10.0%

💡 ตัวอย่างการตั้งค่า:

InpTPSLMethod = 3  // Percentage-Based
InpSLPercentage = 2.0         // SL = 2% ของราคาเข้า
InpTPPercentage = 4.0         // TP = 4% ของราคาเข้า

การคำนวณ: หากเข้า 1.2000, SL = 1.1760, TP = 1.2480

🔀 Hybrid Method (ผสมผสาน ATR + Swing)

เหมาะสำหรับมืออาชีพ

คำอธิบาย: ใช้ ATR สำหรับกำหนด Stop Loss (ปรับตามความผันผวน) และ Swing Points สำหรับกำหนด Take Profit (ตามโครงสร้างตลาด)

ข้อดี: ได้ประโยชน์จากทั้งสองวิธี SL ปรับตามความผันผวน TP ตามโครงสร้างตลาด

ข้อเสีย: ซับซ้อน ต้องเข้าใจทั้งสองแนวคิด

🔧 การทำงาน:

  • Stop Loss: ใช้ ATR × Multiplier (ตามพารามิเตอร์ ATR-Based)
  • Take Profit: ใช้ Swing Points (ตามพารามิเตอร์ Swing Points)
ใช้พารามิเตอร์จาก สำหรับ คำอธิบาย
ATR-Based Method Stop Loss InpATRTimeframe, InpATRPeriod, InpATRMultiplierSL
Swing Points Method Take Profit InpSwingTimeframe, InpSwingLookback, InpSwingBufferPercent

💡 ตัวอย่างการตั้งค่า:

InpTPSLMethod = 4  // Hybrid Method

// ATR Settings (สำหรับ SL)
InpATRTimeframe = PERIOD_H4
InpATRPeriod = 14
InpATRMultiplierSL = 2.0

// Swing Settings (สำหรับ TP)
InpSwingTimeframe = PERIOD_H4
InpSwingLookback = 20
InpSwingBufferPercent = 10.0

🔄 โหมดไฮบริด (Hybrid Mode)

EA รองรับ 4 โหมดการทำงานแบบไฮบริด เพื่อความยืดหยุ่นในการใช้สัญญาณ:

HYBRID_CONFIRMATION

โหมดยืนยัน: สัญญาณภายในและภายนอกต้องตรงกันถึงจะเทรด เหมาะสำหรับการเทรดที่ต้องการความแม่นยำสูง

ใช้เมื่อ: ตลาดมี Volatility สูง หรือต้องการลดความเสี่ยง

HYBRID_INTERNAL_PRIORITY

โหมดสัญญาณภายในหลัก: ใช้สัญญาณภายในเป็นหลัก ใช้สัญญาณภายนอกเป็นตัวกรอง

ใช้เมื่อ: เชื่อมั่นในสูตรสัญญาณภายใน หรือสัญญาณภายนอกไม่เสถียร

HYBRID_EXTERNAL_PRIORITY

โหมดสัญญาณภายนอกหลัก: ใช้สัญญาณภายนอกเป็นหลัก ใช้สัญญาณภายในเป็นตัวยืนยัน

ใช้เมื่อ: มีสัญญาณภายนอกที่เชื่อถือได้ เช่น Copy Trade หรือ Signal Provider

HYBRID_INDEPENDENT

โหมดอิสระ: สัญญาณทั้งสองแบบทำงานอิสระจากกัน เพิ่มโอกาสในการเทรด

ใช้เมื่อ: ต้องการเพิ่มความถี่ในการเทรด หรือทดสอบสัญญาณใหม่

🧠 สูตรสัญญาณภายใน (30 สูตร)

📊 กลุ่มสูตรพื้นฐาน (1-10)

1. Double MA Cross

หลักการ: การตัดกันของ Moving Average คู่

พารามิเตอร์: Fast MA, Slow MA

เหมาะกับ: Trend Market, Timeframe H1 ขึ้นไป

2. MA Cross

หลักการ: ราคาตัด Moving Average

พารามิเตอร์: MA Period, MA Method

เหมาะกับ: ทุกตลาด, มือใหม่

3. Support/Resistance Break

หลักการ: การเบรกเส้น Support/Resistance

🔌 ปัญหาการเชื่อมต่อ:

❌ No Connection

  • ตรวจสอบ Internet Connection
  • ตรวจสอบ Firewall/Antivirus
  • เปลี่ยน Server ของโบรกเกอร์
  • รีสตาร์ท MT5

📶 Poor Connection

  • ปิดโปรแกรมที่ใช้ Internet มาก
  • ใช้ Ethernet แทน WiFi
  • ติดต่อ ISP
  • ใช้ VPS สำหรับ Trading

📊 ปัญหา Spread สูง:

สาเหตุ การแก้ไข การป้องกัน
ช่วงข่าวสำคัญ รอให้ Spread กลับมาปกติ เปิด InpBlockTradingOnNewsTime
ช่วงเปิด/ปิดตลาด หลีกเลี่ยงการเทรดช่วงนี้ ตั้งเวลาเทรดให้เหมาะสม
Liquidity ต่ำ เปลี่ยนไปเทรด Major Pairs ใช้ InpMaxSpread Filter
โบรกเกอร์ไม่ดี เปลี่ยนโบรกเกอร์ เลือกโบรกเกอร์ที่มีชื่อเสียง
🎭 ปัญหาการแสดงผลบนชาร์ต

🖥️ การแก้ไขปัญหาการแสดงผล:

👁️ ไม่แสดงข้อมูลบนชาร์ต

สาเหตุ: การตั้งค่าการแสดงผลปิดอยู่

แก้ไข:

InpShowProfitDetails = true InpShowTradingCosts = true InpShowStatistics = true InpCompactMode = false

📱 ข้อมูลแสดงไม่ครบ (มือถือ)

สาเหตุ: หน้าจอเล็ก

แก้ไข:

InpCompactMode = true InpDisplayPosition = TopLeft // ลดข้อมูลที่แสดง InpShowTradingCosts = false

🎨 การปรับแต่งการแสดงผล:

ปัญหา พารามิเตอร์ ค่าแนะนำ
ข้อมูลทับกับกราฟ InpDisplayPosition TopRight หรือ TopLeft
ข้อมูลมากเกินไป InpCompactMode true
อ่านยาก InpFontSize 8-12 (ขึ้นกับหน้าจอ)
สีไม่เหมาะสม InpTextColor White/Black (ตามธีม)
